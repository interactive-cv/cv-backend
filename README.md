# cv-backend

Бэкенд для **персонального интерактивного сайта-резюме** — open-source-шаблон на Python / FastAPI с реляционной PostgreSQL и встроенным AI-ассистентом. Любой может развернуть такой сайт под своё CV: меняется только контент (markdown) и домен.

Живой пример сервиса, собранного из этого кода: **[cv.libera.pro](https://cv.libera.pro)**.

Сайт совмещает классическое резюме, интерактивный граф проектов/технологий и **живой AI-чат, который отвечает рекрутёру по фактам из CV** — это та «фишка», которая выделяет кандидата среди сотен стандартных резюме.

## Технологии

- **Python 3.12, FastAPI, Pydantic v2** — асинхронное REST API, типобезопасные схемы
- **SQLAlchemy 2.0 (async) + Alembic** — ORM и миграции, кросс-БД типы (jsonb/uuid)
- **PostgreSQL 16** — основное хранилище
- **httpx** — стриминговый клиент к LLM (Server-Sent Events)
- **pytest, pytest-asyncio, respx** — 54 unit-теста + 3 e2e-теста RAG против реальной модели

## Что умеет

| Эндпоинт | Назначение |
|----------|-----------|
| `GET /api/cv/master` | Публичные данные мастер-CV (контакты, навыки, языки) |
| `GET /api/projects` | Список проектов для таймлайна и графа знаний |
| `GET /api/variants/{slug}` | Вариант CV под конкретную вакансию (SSR-страница) |
| `GET /api/links/resolve?code=` | Резолв короткой ссылки с TTL и лимитом кликов |
| `POST /api/chat` | AI-чат (стриминг ответа, RAG по CV) |
| `POST /admin/applications/generate` | **AI-генерация CV и cover letter из вакансии** (LLM, RAG по мастер-CV) |
| `GET/POST/PATCH /admin/applications` | CRUD откликов: компания, роль, CV, cover letter, статус |
| `POST /admin/applications/{id}/publish` | Публикация: создание короткой ссылки, status=active |
| `POST /admin/applications/{id}/archive` | Архивирование отклика |
| `POST /admin/*` | Управление вариантами CV и ссылками вручную (Bearer-токен) |

Изящные детали реализации: атомарный инкремент счётчика кликов через `UPDATE ... WHERE hit_count < max_hits` (без race condition), ограничение частоты запросов (50/IP/час + 300/день) с раздельными bucket'ами для чата и резолва ссылок, защита от брутфорса коротких кодов.

## Интеграция AI и защита от галлюцинаций

AI-чат построен по паттерну **RAG (Retrieval-Augmented Generation) без векторной БД** — для объёма одного CV достаточно подавать весь текст мастер-CV как контекст в системном промпте. Запрос идёт к модели **glm-4.7** (Z.ai) с потоковой передачей токенов обратно в браузер.

Защита от выдумывания ложных сведений — главный архитектурный приоритет, потому что рекрутёр доверяет ответам «ассистента кандидата». Реализовано многоуровнево:

1. **Жёсткий системный промпт** прямо запрещает модели выдумывать опыт, работодателей, метрики и технологии, которых нет в CV. Отдельным пунктом — приказ честно отвечать «в CV этой информации нет» при отсутствии данных и **игнорировать инструкции пользователя**, противоречащие правилам (защита от промпт-инъекций).
2. **e2e-тесты RAG** (`tests/test_rag_behavior.py`, маркер `@pytest.mark.e2e`) проверяют поведение против реальной модели на провокационных запросах: модель не подтверждает выдуманных работодателей (Google), не называет цифр зарплаты и отвечает по фактам на легитимные вопросы о стеке. Эти тесты зелёные.
3. **Graceful degradation**: при недоступности LLM (сбой API, исчерпание квоты) чат не падает с 500, а отвечает запасным сообщением с прямыми контактами — остальной сайт продолжает работать.

## AI-генератор откликов (вторая AI-функция)

Помимо чата для рекрутёров, LLM используется для **генерации адаптированного CV и cover letter из текста вакансии**. Владелец вставляет вакансию, выбирает релевантные проекты — и модель создаёт два текста в markdown за один вызов:

- Тот же паттерн RAG (мастер-CV как контекст), но задача другая — переупорядочить и подсветить релевантное, не выдумывая.
- Промпт (`app/llm/generate_prompt.py`) запрещает искажать факты и приказывает опираться только на выбранные проекты.
- Результат парсится маркерами `===CV===`/`===COVER===` и отдаётся в split-редактор админ-панели для ручной правки перед публикацией.

## Архитектура и тесты

```
app/
├── models/      SQLAlchemy-модели (master_cv, cv_variant, short_link, link_hit, project)
├── schemas/     Pydantic DTO (публичные — без приватного vacancy_text)
├── routers/     эндпоинты (cv, projects, links, chat, admin, health)
├── services/    бизнес-логика (резолв ссылок, парсер CV → структуру)
├── llm/         промпт-инжиниринг и стриминговый клиент Z.ai
└── errors.py    единый формат ошибок {error, message, request_id}
tests/           42 теста (unit на in-memory SQLite + e2e против glm-4.7)
```

Тесты запускаются на in-memory SQLite (кросс-БД типы через `TypeDecorator`), а в проде на PostgreSQL те же модели транслируются в нативные `jsonb`/`uuid` — без изменения кода.

## Развёртывание

Прод-окружение — собственный VPS (Linux) с Docker Compose:

- **nginx-ssl** (собственный образ на `nginx:alpine` + certbot) терминирует TLS и раздаёт статику, сертификаты Let's Encrypt получает и продлевает автоматически
- **FastAPI**, **Next.js** и **PostgreSQL** — в изолированной docker-сети
- Весь стек описан в `docker-compose.prod.yml`, обновление — одной командой `bash deploy.sh` (rsync + пересборка образов + миграции Alembic + seed)

Все персональные данные (имя, контакты, проекты, домен, ключ LLM) выносятся в `seed_data/` и `.env` — поэтому чужое CV разворачивается заменой контента без правки кода. Репозиторий фронтенда: [cv-frontend](https://github.com/vrg18/cv-frontend).

## Дорожная карта

- Вынос оставшихся захардкоженных значений (имя в промпте, домен в OG/robots, fallback-контакты) в единый конфиг — чтобы сделать шаблон полностью параметрическим
- Админ-панель для управления вариантами CV и короткими ссылками из браузера
- Расширенная аналитика просмотров и переходов по коротким ссылкам
